# Remove Stored Transcript Path from Checkpoint Metadata

## Context

When running `entire resume`, the CLI restores session transcripts from checkpoints. Currently, checkpoint metadata stores a `transcript_path` field that embeds the original repository location, causing restoration to fail when the project moves.

## The Problem

Agents (Claude Code, Gemini CLI) store transcripts in directories derived from the repository path:
- Claude: `~/.claude/projects/-Users-alice-projects-cli/session.jsonl`
- Gemini: `~/.gemini/tmp/-Users-alice-projects-cli/chats/session.json`

The "project directory" component (`-Users-alice-projects-cli`) is created by sanitizing the repo absolute path (`/Users/alice/projects/cli`), replacing non-alphanumeric characters with dashes.

**When a project moves:**
1. Checkpoint created on machine A: `/Users/alice/projects/cli`
2. Transcript stored at: `~/.claude/projects/-Users-alice-projects-cli/session.jsonl`
3. Checkpoint metadata saves: `.claude/projects/-Users-alice-projects-cli/session.jsonl`
4. Project cloned on machine B: `/Users/bob/work/cli`
5. Agent now stores transcripts at: `~/.claude/projects/-Users-bob-work-cli/session.jsonl`
6. Resume tries to restore to: `~/.claude/projects/-Users-alice-projects-cli/session.jsonl` (WRONG!)

This breaks cross-machine portability and even moving repos on the same machine.

## Root Cause Analysis

**Two different TranscriptPath fields:**

1. **`session.State.TranscriptPath`** (runtime session state in `.git/entire-sessions/`):
   - Points to the **live transcript file** during an active session
   - Used for mid-session commit detection (when user commits before calling `Stop`)
   - Updated on each turn from the agent's hook input
   - **This one IS needed** - operational data for tracking active sessions

2. **`CommittedMetadata.TranscriptPath`** (checkpoint metadata on `entire/checkpoints/v1` branch):
   - Stored as home-relative path (e.g., `.claude/projects/-Users-alice-projects-cli/`)
   - Intended for resume to restore transcript to original location
   - **This one causes the bug** - embeds the sanitized repo path from checkpoint creation time

**The stored path is unnecessary because:**
- We can always compute the correct transcript path from the current repo location using `agent.GetSessionDir(repoRoot)`
- The computed path automatically handles repo moves/clones
- Even agents with "complex" path schemes (like Gemini's hashing) still derive paths from the repo location, so they have the same problem

## Solution

**Stop storing `transcript_path` in checkpoint metadata entirely.** Always compute the transcript path at restore time based on the current repo location.

### Implementation

1. **Remove `SessionTranscriptPath` from checkpoint writes** (`manual_commit_condensation.go`)
   - Delete line that sets `SessionTranscriptPath: homeRelativePath(state.TranscriptPath)`
   - This stops writing the field to new checkpoints

2. **Update `RestoreLogsOnly()` to always compute transcript path** (`manual_commit_rewind.go`)
   - Remove `resolveTranscriptPathFromMetadata()` call
   - Always use `agent.GetSessionDir(repoRoot)` to compute session directory from current repo location
   - Use `ResolveSessionFilePath()` to construct the full transcript path
   - This makes restoration location-independent

3. **Keep struct fields for backward compatibility**
   - Keep `CommittedMetadata.TranscriptPath` field and JSON tag (old checkpoints may have it)
   - Keep `WriteCommittedOptions.SessionTranscriptPath` field (not used anymore but avoids breaking API)
   - Mark as deprecated in comments

4. **Update auto-commit strategy similarly**
   - Remove `TranscriptPath` writes in auto-commit condensation
   - Update any restoration logic if needed

### Why This Works

- **Always uses current repo location**: `agent.GetSessionDir(repoRoot)` computes where the agent stores transcripts based on the CURRENT repo path
- **Handles all repo moves**: Whether the repo moves on the same machine or is cloned to a different machine
- **Simpler logic**: No fallback path resolution, no special cases
- **Backward compatible**: Old checkpoints with `transcript_path` are ignored (we just compute the correct path anyway)
