okay let's update soem comments:

agent.Detect() iterates over the internal registry map, so if multiple agents report presence (e.g., both .claude/ and .gemini/ exist) the detected agent is nondeterministic. That can lead to installing hooks for the wrong agent. Consider making detection deterministic (iterate agent.List() and check DetectPresence() in a stable order) and/or handling the "multiple agents detected" case by prompting the user instead of picking an arbitrary one.
