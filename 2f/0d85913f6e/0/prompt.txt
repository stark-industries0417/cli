# Detect External Hook Managers and Warn Users

## Context

PR #355 fixed hook installation to use `git rev-parse --git-path hooks`, which correctly resolves `core.hooksPath` (e.g., Husky's `.husky/_/`). However, Husky regenerates `.husky/_/` on every `npm install` (via the `prepare` script), so Entire's hooks get overwritten.

**Goal:** Detect when an external hook manager is in use, warn the user during `entire enable`, and provide copy-paste instructions for a durable integration.

## New Files

### `cmd/entire/cli/strategy/hook_managers.go`

**Data model:**
```go
type HookManager struct {
    Name       string // "Husky", "Lefthook", "pre-commit", "Overcommit"
    ConfigPath string // relative path that triggered detection (e.g., ".husky/")
    OverwritesHooks bool // true if tool will overwrite Entire's hooks on reinstall
}
```

**Detection function:**
```go
func DetectHookManagers(repoRoot string) []HookManager
```

Detection is filesystem-only (no git commands, no file reads — just `os.Stat`):

| Manager | Signal | `OverwritesHooks` |
|---------|--------|-------------------|
| Husky v9+ | `.husky/_/` directory exists | `true` |
| Lefthook | `lefthook.yml` or `.lefthook.yml` exists | `false` |
| pre-commit | `.pre-commit-config.yaml` exists | `false` |
| Overcommit | `.overcommit.yml` exists | `false` |

Returns a slice (repos can have multiple managers, e.g., Husky + lint-staged config).

**Warning generation:**
```go
func HookManagerWarning(managers []HookManager, cmdPrefix string) string
```

Builds warning text from `buildHookSpecs(cmdPrefix)` via an `extractCommandLine()` helper — this keeps the copy-paste instructions always in sync with actual hook content.

**Public entry point (called from setup.go):**
```go
func CheckAndWarnHookManagers(w io.Writer) []HookManager
```

Calls `paths.RepoRoot()`, `DetectHookManagers()`, writes warning to `w`, returns detected managers.

## Warning Messages

**Husky (overwrites hooks):**
```
Warning: Husky detected (.husky/)

  Husky may overwrite hooks installed by Entire on npm install.
  To make Entire hooks permanent, add these lines to your Husky hook files:

    .husky/prepare-commit-msg:
      entire hooks git prepare-commit-msg "$1" "$2" 2>/dev/null || true

    .husky/commit-msg:
      entire hooks git commit-msg "$1" || exit 1

    .husky/post-commit:
      entire hooks git post-commit 2>/dev/null || true

    .husky/pre-push:
      entire hooks git pre-push "$1" || true
```

**Category B tools (backup/chain works, but may need re-enable):**
```
Note: Lefthook detected (lefthook.yml)

  If Lefthook reinstalls hooks, run 'entire enable' to restore Entire's hooks.
```
